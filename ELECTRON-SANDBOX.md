# Electron 沙盒问题解决方案

## 问题描述

在macOS上运行Electron应用时遇到沙盒权限问题，导致无法创建缓存和GPU进程。

## 解决方案

### 方案1：使用浏览器版本（推荐）

1. 启动开发服务器：
   ```bash
   npm run dev
   ```

2. 在浏览器中打开：
   ```
   http://localhost:3000
   ```

### 方案2：修改系统权限

1. 打开"系统偏好设置" > "安全性与隐私"
2. 点击"隐私"标签
3. 在左侧选择"完全磁盘访问权限"
4. 添加Electron应用并允许访问
5. 重启应用

### 方案3：使用代码签名（生产环境）

1. 获取Apple开发者账号
2. 对应用进行代码签名
3. 使用签名后的应用

### 方案4：禁用沙盒（临时）

在electron-simple.js中已经尝试禁用沙盒，但macOS仍然强制启用。

## 当前状态

✅ Web版本：完全正常工作  
⚠️ Electron版本：需要系统权限调整

## 使用建议

1. **开发阶段**：使用浏览器版本
2. **生产环境**：考虑代码签名或PWA方案
3. **跨平台需求**：Windows/Linux版本可能没有这个问题

## 技术细节

macOS的沙盒机制非常严格，即使代码中设置`sandbox: false`，系统仍可能强制启用沙盒。这是Apple的安全机制，旨在防止恶意软件访问系统资源。

## 替代方案

如果必须使用桌面应用，可以考虑：

1. **PWA（渐进式Web应用）**：
   - 可安装到桌面
   - 无沙盒限制
   - 接近原生体验

2. **Tauri**：
   - 更轻量的桌面应用框架
   - 更好的权限管理
   - 需要Rust环境

3. **Capacitor**：
   - 移动优先的框架
   - 支持桌面部署
